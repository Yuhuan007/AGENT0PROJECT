## 数据流图

```
用户输入: "请计算100+200，然后查询北京天气"
    │
    ▼
┌───────────────────────────────────────────────┐
│ Session Manager                                │
│  - 创建/获取会话                               │
│  - 添加用户消息到历史                         │
└────────────────┬──────────────────────────────┘
                 │
                 ▼
┌───────────────────────────────────────────────┐
│ Step 1: Perception                             │
│  - session.get_context(max_turns=10)           │
│  - 构建messages列表                            │
│  - 添加system prompt（含工具Schema）          │
└────────────────┬──────────────────────────────┘
                 │
                 ▼
┌───────────────────────────────────────────────┐
│ Step 2: Planning                               │
│  - llm.invoke(messages)                        │
│  - LLM输出:                                    │
│    [TOOL_CALL]                                 │
│    {"tool": "calculator",                      │
│     "arguments": {"expression": "100+200"}}    │
│    [/TOOL_CALL]                                │
└────────────────┬──────────────────────────────┘
                 │
                 ▼
┌───────────────────────────────────────────────┐
│ Step 3: Action                                 │
│  - _parse_tool_calls(llm_output)               │
│  - tool_registry.execute_tool(...)             │
│  - 结果: "计算结果: 300"                       │
│  - 保存到session (assistant + tool)           │
└────────────────┬──────────────────────────────┘
                 │
                 ▼
┌───────────────────────────────────────────────┐
│ Step 4: Reflection                             │
│  - 检查: 有工具调用 → 继续循环                │
│  - 进入下一轮 (Iteration 2)                   │
└────────────────┬──────────────────────────────┘
                 │
                 ▼
         重复 Perception → Planning
                 │
                 ▼
         Planning输出:
         [TOOL_CALL]
         {"tool": "weather",
          "arguments": {"location": "北京"}}
         [/TOOL_CALL]
                 │
                 ▼
         Action执行:
         weather("北京") → "晴朗, 15°C"
                 │
                 ▼
         Reflection检查:
         所有任务完成 → 生成最终回答
                 │
                 ▼
┌───────────────────────────────────────────────┐
│ 最终输出                                       │
│ "计算结果是300。北京今天天气晴朗，温度15°C。" │
└───────────────────────────────────────────────┘
```


## 工具注册与调用流程

```
开发者定义工具
    │
    ├─ 编写工具函数: calculator_tool(expression)
    │
    ├─ 定义Schema:
    │  {
    │    "type": "object",
    │    "properties": {
    │      "expression": {"type": "string", "description": "..."}
    │    },
    │    "required": ["expression"]
    │  }
    │
    ▼
注册到ToolRegistry
    │
    └─ tool_registry.register(
         name="calculator",
         description="执行数学计算",
         parameters=CALCULATOR_SCHEMA,
         function=calculator_tool
       )
    │
    ▼
Schema被注入到System Prompt
    │
    ├─ LLM看到: "可用工具: calculator - 执行数学计算"
    ├─ 参数定义: {"expression": "数学表达式"}
    │
    ▼
LLM自主决策
    │
    ├─ 理解用户需求
    ├─ 判断是否需要工具
    ├─ 选择合适的工具
    ├─ 生成工具调用JSON
    │
    ▼
Runtime解析并执行
    │
    ├─ _parse_tool_calls() → 提取JSON
    ├─ tool_registry.execute_tool() → 调用函数
    ├─ 获得结果
    │
    ▼
结果注入上下文
    │
    └─ 下一轮LLM调用时可见
```


## Session与Context管理

```
┌─────────────────────────────────────────────┐
│ SessionManager                               │
│                                              │
│  sessions = {                                │
│    "session-001": Session(...),             │
│    "session-002": Session(...),             │
│    ...                                       │
│  }                                           │
│                                              │
│  每个Session独立：                           │
│  - session_id: 唯一标识                     │
│  - messages: List[Message]                  │
│  - metadata: Dict                           │
│  - created_at: datetime                     │
│                                              │
└─────────────────────────────────────────────┘

Session.get_context(max_turns=10):
    │
    ├─ 获取messages列表
    │
    ├─ 计算: 10轮 = 20条消息 (user + assistant)
    │
    ├─ 截取: messages[-20:]
    │
    ├─ 转换格式:
    │  [
    │    {"role": "user", "content": "..."},
    │    {"role": "assistant", "content": "..."},
    │    {"role": "tool", "content": "..."},
    │    ...
    │  ]
    │
    └─ 返回给Runtime
```


## 错误处理与鲁棒性

```
1. 最大迭代保护
   ┌─────────────┐
   │ while i < 5 │ ──► 防止死循环
   └─────────────┘

2. 工具调用失败
   ┌──────────────────┐
   │ try:             │
   │   tool.function()│
   │ except:          │ ──► 返回错误信息，不中断
   │   return "❌ ..." │
   └──────────────────┘

3. JSON解析容错
   ┌──────────────────┐
   │ try:             │
   │   json.loads()   │
   │ except:          │ ──► 跳过该工具调用
   │   continue       │
   └──────────────────┘

4. Context自动截断
   ┌──────────────────┐
   │ messages[-20:]   │ ──► 防止token溢出
   └──────────────────┘

5. Session隔离
   ┌──────────────────┐
   │ 不同session_id   │ ──► 会话间互不影响
   │ → 独立context    │
   └──────────────────┘
```


## 可扩展性设计

```
1. 新增工具
   ┌────────────────────────────────┐
   │ def my_new_tool(arg1, arg2):   │
   │     # 实现逻辑                 │
   │     return result              │
   │                                │
   │ SCHEMA = {...}                 │
   │                                │
   │ tool_registry.register(...)    │
   └────────────────────────────────┘
   ✅ 无需修改核心代码

2. 替换LLM
   ┌────────────────────────────────┐
   │ class MyCustomLLM:             │
   │     def invoke(self, messages):│
   │         # 自定义实现           │
   │         return response        │
   └────────────────────────────────┘
   ✅ 只需实现invoke接口

3. 持久化存储
   ┌────────────────────────────────┐
   │ class DatabaseSessionManager:  │
   │     def create_session():      │
   │         # 写入数据库           │
   │     def get_session():         │
   │         # 从数据库读取         │
   └────────────────────────────────┘
   ✅ 替换SessionManager即可

4. 流式输出
   ┌────────────────────────────────┐
   │ def run_stream(self, ...):     │
   │     for chunk in llm.stream(): │
   │         yield chunk            │
   └────────────────────────────────┘
   ✅ 添加新方法，不影响原有功能

5. 并行工具调用
   ┌────────────────────────────────┐
   │ import asyncio                 │
   │ async def execute_parallel():  │
   │     tasks = [...]              │
   │     results = await gather()   │
   └────────────────────────────────┘
   ✅ 在Action步骤中增强
```


## 对比其他框架

| 特性 | MiniAgentRuntime | LangGraph | AutoGen |
|------|------------------|-----------|---------|
| 代码量 | ~400行 | ~10000行+ | ~15000行+ |
| 学习曲线 | 平缓 | 陡峭 | 陡峭 |
| 依赖项 | 极少 | 中等 | 较多 |
| 可定制性 | 高 | 中 | 中 |
| 适用场景 | 学习/原型 | 生产环境 | 多Agent系统 |
| 工具调用 | 自定义协议 | Function Calling | Function Calling |
| 会话管理 | 内置 | 需自行实现 | 内置 |

**优势**：
- ✅ 代码简洁，易于理解
- ✅ 无框架绑定，自由度高
- ✅ 轻量级，快速启动
- ✅ 适合教学和快速原型

**局限**：
- ⚠️ 不适合大规模生产
- ⚠️ 缺少高级特性（流程图、多Agent协调等）
- ⚠️ 需要手动处理更多细节
